What Are Models?

OpenClaw doesn't bundle its own AI — it's a routing layer. You pick a model by provider/model ref, and the gateway sends your messages there.

anthropic/claude-opus-4-6
→
openai/gpt-5.5
→
google/gemini-3.1-pro
→
ollama/llama3.3

Every message goes through exactly one model. You can switch mid-conversation.
Default lives in config: agents.defaults.model.primary

openclaw models set anthropic/claude-opus-4-6

Selection Hierarchy

OpenClaw picks a model in a clear chain of preference.

① Primary Model
agents.defaults.model.primary
↓
② Fallbacks
agents.defaults.model.fallbacks (ordered list)
↓
③ Auth Profile Rotation
Multiple API keys / OAuth profiles per provider
🎯 Configured Default
Normal starting point. Uses configured fallback chain. Auto-recovery probes the primary periodically.
👤 User Override
/model or picker selection. Strict — won't silently fall back to another model if unreachable.
🤖 Auto Fallback
Runtime selects a fallback after primary failure. Persists per session so it doesn't retry-fail every turn.

Failure & Failover Flow

When a model call fails, OpenClaw doesn't just give up — it walks a smart recovery path.

Primary Fails
→
Rotate Auth Profiles
(rate limit → next key)
→
Cooldown & Backoff
(1m → 5m → 25m → 1h)
→
Next Fallback Model
🔁 Rate Limits
429s, concurrency limits, quota exhaustion → rotate keys/profiles first, exponential backoff, then fallback models.
💳 Billing Failures
Insufficient credits → longer backoff (5h, doubling to 24h cap). Profile disabled until reset time passes.
⏱️ Timeouts
Overloaded provider, no response → treated as failover-worthy. One rotation then move on.
🔍 Periodic Recovery Probe
Every 5 min OpenClaw probes the original primary. If it recovers, auto-clears the fallback and announces it.

Switching Models Live

Change the model mid-conversation without restarting the gateway. Persists per session.

/model
Interactive Picker
Shows a numbered list of available models. Type /model 3 to select the third one.
Direct Selection
/model openai/gpt-5.4 — exact ref or alias. Persists immediately.
/model list
Provider Discovery
/model list shows all available models. /models openai shows that provider's catalog.
Status & Pinning
/model status shows active model, auth status, fallback state. User-selections are pinned and strict.
⚡ Live Switching
If the agent is idle, the new model takes effect immediately. If a run is in progress, OpenClaw queues the switch for the next clean retry point — no dropped messages.

Auth Profiles & Rotation

Multiple keys and OAuth accounts per provider. Rotated on rate limits, stickied per session for cache warmth.

🔑 API Keys
Set via env vars (ANTHROPIC_API_KEY, OPENAI_API_KEY, etc.) or auth.profiles. Supports numbered keys for rotation: OPENAI_API_KEY_1, OPENAI_API_KEY_2.
🔐 OAuth
OpenAI ChatGPT/Codex, xAI SuperGrok, Gemini CLI, Qwen OAuth, MiniMax OAuth. Stored as profiles in the agent SQLite store.
🔄 Rotation Order
OAuth profiles tried before API keys. Within each type: least-recently-used first. Cooldown profiles moved to the end.
📌 Session Stickiness
Same profile reused within a session for cache continuity. Cleared on /new, /reset, or compaction. Manual /model …@profileId is strict.
# Configure rotation priority openclaw config set auth.order '{"openai":["openai:user@example.com","openai:api-key-backup"]}' --merge

Model Scanning

openclaw models scan probes OpenRouter's free model catalog and builds ranked recommendations.

📡 What It Does
Fetches all free models from OpenRouter, filters by your criteria (--min-params, --max-age-days, --provider), then live-probes each for tool support and image capability.
📊 Ranking
1. Image support
2. Tool latency
3. Context size
4. Parameter count
openclaw models scan \ --min-params 70 \ --max-age-days 60 \ --max-candidates 5 \ --set-default
⚙️ Key Flags
--no-probe — metadata only, no key needed
--set-default — assigns best pick as primary
--set-image — best image model as imageModel

Model Allowlisting

Control which models users can pick with agents.defaults.models — the allowlist.

🔒 Exact Entries
Add specific provider/model pairs. Only those are available. Picker won't show anything else.
🌐 Provider Wildcards
"openai/*": {} — allows any model from that provider. New models appear automatically without config edits.
openclaw config set agents.defaults.models \ '{ "anthropic/claude-sonnet-4-6": {"alias":"Sonnet"}, "anthropic/*": {}, "ollama/*": {} }' --strict-json --merge
⚠️ Unlisted Models
If a user tries a model outside the allowlist, OpenClaw returns: "Model is not allowed" — the session won't reply until a valid model is chosen or the allowlist is updated.
