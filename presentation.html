🧠 The Big Idea

OpenClaw is a personal AI agent runtime — a single daemon that connects your AI brain to all your messaging channels, devices, and tools.

You Own It
Runs on your hardware. Your config. Your data. Your model keys. No cloud dependency.
🔌
Universal Connector
WhatsApp, Telegram, Discord, Signal, iMessage, Slack — one agent talks to all of them.

Plug-in Architecture

No hidden state. Your agent remembers by writing plain Markdown to disk. Transparent by design.

👥 One Gateway, Many Brains

Multi-agent routing = isolated personas in one process

🧑‍💼
Agent = Scoped Brain
Each agent gets its own workspace, SOUL.md, memory, auth profiles, and session store. No cross-talk.
agents: { list: [ { id: "chat", workspace: "~/workspace-chat" }, { id: "opus", workspace: "~/workspace-opus" } ] }
🔀
Bindings Route Messages
Deterministic rules map (channel, account, peer) → agent. Most specific match wins.
bindings: [ { agentId: "opus", match: { channel: "whatsapp", peer: { kind: "direct", id: "+1555..." } } }, { agentId: "chat", match: { channel: "whatsapp", accountId: "*" } } ]
🔗
Per-Channel Accounts
Multiple WhatsApp numbers, Discord bots, Telegram bots — each account routes to its own agent.
🛡️
Per-Agent Sandboxing
Restrict tools per agent. One can be full-access, another sandboxed with read-only tools.

📝 The Markdown Files — Your Agent's Brain

Everything the agent knows is written to disk. Here's what each file does.

AGENTS.md
Operating instructions, rules, priorities. The agent's "how to behave" manual. Loaded every session.
SOUL.md
Personality, tone, boundaries. Give your agent an actual voice — sharp, blunt, funny, whatever fits.
USER.md
Who you are. Name, pronouns, timezone, preferences. "Commander" for this session.
IDENTITY.md
Agent name, vibe, emoji. "QWERT.SYS" - the digital persona users interact with.
TOOLS.md
Local tool notes (SSH hosts, camera names, TTS voices). Doesn't enable/disable tools — just guidance.
HEARTBEAT.md
Tiny periodic checklist. Keeps the agent checking email, calendar, weather without token burn.
MEMORY.md
Curated long-term memory. Durable facts, decisions, summaries. Distilled from daily notes.
memory/
Daily logs: YYYY-MM-DD.md. Raw observations, session summaries, indexed for search.
BOOTSTRAP.md
One-time first-run ritual. Created for brand-new workspaces, deleted after completion.
BOOT.md
Optional startup checklist. Runs automatically on gateway restart via hooks.
DREAMS.md
Dreaming diary — background memory consolidation phases, promotion candidates.
skills/
Workspace skills. Highest-precedence skill location. Each skill = SKILL.md + assets.

🗂️ Some live under the workspace. Config, auth, and sessions live under ~/.openclaw/ — separate, never committed.

🎭 SOUL.md + MEMORY.md = Personality + Continuity

The two most important files in your workspace

🎭
SOUL.md — The Voice
  • Tone, opinions, brevity, humor
  • Boundaries (when to say no)
  • "Never open with Great question!"
  • Be the assistant you'd want at 2am

Injected every DM session. Has real weight in the system prompt.

🧠
MEMORY.md — The Archive
  • Durable facts: preferences, decisions
  • Curated — distilled from daily logs
  • Loaded at start of private sessions
  • Action-sensitive: notes on when to act

Memory flush runs before compaction to save important context automatically.

SOUL.md = voice MEMORY.md = facts memory/*.md = raw logs AGENTS.md = rules

🔧 The Power Tools

Hooks, Skills, Compaction, and Commitments

🪝
Hooks
Event-driven scripts at every lifecycle stage: message_received, before_tool_call, agent:bootstrap, gateway_start, and more.
🧩
Skills
Reusable procedure files (SKILL.md) loaded from workspace, agent, managed, or bundled dirs. Apps, plugins, shared.
📦
Compaction
Summarizes old conversation history when the context window gets full. Memory flush saves key context first.
⏰
Commitments
Inferred follow-ups. "I have an interview tomorrow" → agent checks in later. Opt-in, short-lived.
⏲️
Cron
Exact scheduled tasks: "remind me at 3PM", "run this report every weekday". Precise timing.
🎨
Canvas
Agent-editable HTML/CSS/JS served from the gateway. Live interactive UIs nodes can display.
